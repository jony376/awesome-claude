The `/debug` command provides intelligent debugging assistance with root cause analysis, systematic troubleshooting, performance profiling, and automated fix generation across common languages and frameworks.

## Usage

```
/debug [options] <error_or_file>
```

## Options

### Debug modes

- `--interactive` ‚Äî step-by-step guided debugging
- `--automated` ‚Äî automatic issue detection and fixes
- `--analysis` ‚Äî deep code analysis without applying fixes
- `--performance` ‚Äî performance bottleneck identification

### Error types

- `--runtime` ‚Äî runtime errors and exceptions
- `--logic` ‚Äî logic errors and unexpected behavior
- `--memory` ‚Äî memory leaks and allocation issues
- `--network` ‚Äî network and API problems
- `--database` ‚Äî database connection and query issues

### Scope

- `--function` ‚Äî debug a specific function
- `--class` ‚Äî debug an entire class
- `--module` ‚Äî debug a module or file
- `--system` ‚Äî system-wide debugging

### Output formats

- `--format=detailed` ‚Äî comprehensive analysis (default)
- `--format=quick` ‚Äî quick diagnosis and fix
- `--format=checklist` ‚Äî debugging checklist
- `--format=trace` ‚Äî execution-trace analysis

## Example

```
/debug --runtime --format=detailed src/user-manager.js
```

For each issue it finds ‚Äî missing input validation, race conditions in ID generation, unhandled async errors, case-sensitive lookups, or mutating an array while iterating ‚Äî the assistant reports the location and severity, explains the root cause, and proposes a concrete fix with a prevention strategy.
