## Source Verification Notes

Verified against official Claude Code documentation on **2026-06-14**:

- Official plugin-dependencies docs describe semver constraints in plugin.json dependencies arrays to prevent surprise upstream breakage.
- Docs require git tag naming {plugin-name}--v{version} for constraint resolution and document claude plugin tag --push.
- Cross-marketplace dependencies require allowCrossMarketplaceDependenciesOn in marketplace.json, which this pack verifies during governance review.
- Docs document range-conflict, dependency-version-unsatisfied, and no-matching-tag errors surfaced in claude plugin list and /doctor.
- Auto-update resolves constrained dependencies to the highest tag satisfying all installed plugins' ranges rather than latest marketplace head.

## Troubleshooting

**Issue:** Official docs disagree with local behavior
**Fix:** Check Claude Code version and changelog; reproduce on a clean staging project.

**Issue:** Managed policy blocks intended workflow
**Fix:** Escalate policy change or redesign workflow to approved surfaces.

**Issue:** Integration auth fails intermittently
**Fix:** Refresh OAuth tokens, clock skew, and redirect URLs per MCP or SDK docs.

**Issue:** Autonomous run caused unexpected edits
**Fix:** Narrow tool allowlists, add hooks, and require human approval for writes.
