## Installation
1. Create hooks directory: mkdir -p .claude/hooks
2. Create hook file: touch .claude/hooks/performance-monitor.sh
3. Make executable: chmod +x .claude/hooks/performance-monitor.sh
4. Add configuration from Hook Configuration section above to .claude/settings.json or ~/.claude/settings.json
5. Alternative: Use the interactive /hooks command in Claude Code

### Config paths

- Local (not committed): `.claude/settings.local.json`
- User settings (global): `~/.claude/settings.json`
- Project-wide (committed): `.claude/settings.json`

## Hook Script
```bash
#!/bin/bash
echo "🔍 Performance Monitor - Analyzing system performance..."
echo "📊 Monitoring Areas:"
echo "  • Application Performance Metrics"
echo "  • Database Performance"
echo "  • Frontend Performance (Web Vitals)"
echo "  • Infrastructure Monitoring"
if command -v node >/dev/null 2>&1; then
  echo "✓ Node.js available for performance profiling"
fi
if command -v lighthouse >/dev/null 2>&1; then
  echo "✓ Lighthouse available for web performance"
fi
if command -v artillery >/dev/null 2>&1; then
  echo "✓ Artillery available for load testing"
fi
echo "💻 System Performance:"
if command -v free >/dev/null 2>&1; then
  mem_usage=$(free | grep Mem | awk '{printf "%.2f", $3/$2 * 100.0}')
  echo "  Memory Usage: ${mem_usage}%"
fi
disk_usage=$(df / | tail -1 | awk '{print $5}' | sed 's/%//')
echo "  Disk Usage: ${disk_usage}%"
echo "💡 Performance Recommendations:"
echo "  • Implement response time tracking for API endpoints"
echo "  • Set up Web Vitals monitoring for frontend performance"
echo "  • Monitor database query performance"
echo "  • Set up automated performance testing"
echo "🎯 Performance monitor analysis complete!"
exit 0
```

### System Resource Monitoring

Enhanced hook script for cross-platform system resource monitoring

```bash
#!/bin/bash
echo "💻 System Performance:"
if command -v free >/dev/null 2>&1; then
  mem_usage=$(free | grep Mem | awk '{printf "%.2f", $3/$2 * 100.0}')
  echo "  Memory Usage: ${mem_usage}%"
  if (( $(echo "$mem_usage > 85" | bc -l) 2>/dev/null )); then
    echo "  🚨 High memory usage detected!"
  fi
elif command -v vm_stat >/dev/null 2>&1; then
  echo "  Memory Usage: $(vm_stat | grep 'Pages active' | awk '{print $3}' | sed 's/\.//')"
fi
disk_usage=$(df / | tail -1 | awk '{print $5}' | sed 's/%//')
echo "  Disk Usage: ${disk_usage}%"
if (( disk_usage > 90 )); then
  echo "  🚨 High disk usage detected!"
fi
if command -v uptime >/dev/null 2>&1; then
  load_avg=$(uptime | awk -F'load average:' '{print $2}' | awk '{print $1}' | sed 's/,//')
  cpu_cores=$(sysctl -n hw.ncpu 2>/dev/null || nproc 2>/dev/null || echo "1")
  load_percent=$(echo "scale=1; $load_avg * 100 / $cpu_cores" | bc -l 2>/dev/null || echo "0")
  echo "  Load Average: $load_avg (${load_percent}% of ${cpu_cores} cores)"
fi
exit 0
```

### Web Vitals Tracking with Lighthouse

Enhanced hook script for Web Vitals tracking using Lighthouse

```bash
#!/bin/bash
if command -v lighthouse >/dev/null 2>&1; then
  if curl -s http://localhost:3000 >/dev/null 2>&1; then
    echo "🌐 Running Lighthouse Web Vitals analysis..."
    lighthouse http://localhost:3000 --output json --quiet --chrome-flags='--headless --no-sandbox' > /tmp/lighthouse.json 2>&1
    if [ -f /tmp/lighthouse.json ]; then
      LCP=$(jq -r '.audits["largest-contentful-paint"].numericValue' /tmp/lighthouse.json 2>/dev/null || echo "0")
      CLS=$(jq -r '.audits["cumulative-layout-shift"].numericValue' /tmp/lighthouse.json 2>/dev/null || echo "0")
      FCP=$(jq -r '.audits["first-contentful-paint"].numericValue' /tmp/lighthouse.json 2>/dev/null || echo "0")
      echo "  📊 LCP: ${LCP}ms"
      echo "  📊 CLS: $CLS"
      echo "  📊 FCP: ${FCP}ms"
    fi
    rm -f /tmp/lighthouse.json
  fi
fi
exit 0
```

## Troubleshooting
### Memory usage calculation fails on macOS systems

Hook uses free command which is Linux-only. On macOS, install free via brew install free or modify hook to use vm_stat | grep 'Pages active' for memory statistics instead. Verify system commands. Test with various operating systems.

### bc command not found error on minimal systems

Install bc for floating-point arithmetic (apt-get install bc or brew install bc). Alternatively, modify memory comparison to use integer math with awk instead of bc -l for percentage checks. Verify bc installation. Test with various system configurations.

### Lighthouse or Artillery tools not detected

Install globally with npm install -g lighthouse @artillery/core or install locally and modify hook to check npx lighthouse and npx artillery instead of direct commands. Verify tool installation. Test with various tool configurations.

### Hook shows same performance data every run

This is a stop hook providing system snapshot at session end. For continuous monitoring, integrate with dedicated APM tools or add timestamped logging to track changes over multiple sessions. Verify hook execution timing. Test with various session durations.

### Load average shows very high values but system responsive

Load average is relative to CPU core count. Value of 8 is normal for 8-core system. Divide load average by core count (sysctl -n hw.ncpu on macOS, nproc on Linux) for actual load percentage. Verify CPU core count. Test with various system configurations.

### Web Vitals tracking fails with Chrome not found

Install Chrome/Chromium: brew install chromium on macOS or apt-get install chromium-browser on Linux. Set CHROME_PATH environment variable. Use --chrome-flags='--headless --no-sandbox' for CI environments. Verify Chrome installation. Test with various Chrome configurations.

### Bundle size analysis tools not available

Install bundle analyzers: npm install --save-dev webpack-bundle-analyzer for webpack, npm install --save-dev vite-bundle-visualizer for Vite. Verify build output directories exist. Ensure build is completed before analysis. Verify tool installation. Test with various build configurations.

### Performance monitoring overhead affects system performance

Reduce monitoring frequency or scope. Use lightweight monitoring tools. Add conditional execution based on system load. Consider sampling instead of continuous monitoring. Verify monitoring overhead. Test with various system loads.
